# OpenQuant 数据质量说明

## 一、数据准确性

| 数据类型 | 数据源 | 准确性 | 说明 |
|---------|--------|--------|------|
| **日线行情** | 东方财富 / 新浪 / 腾讯 | 高 | 来自正规财经平台，与交易所数据一致 |
| **基本面** | 东方财富 | 高 | 财报、市值等公开数据 |
| **新闻** | 东方财富 | 中 | 爬取公开新闻，可能有延迟 |
| **实时行情** | 东方财富 / 新浪 | 中 | 免费接口，非交易所直连 |

**结论**：日线数据可信，适合策略回测与 AI 分析；实时数据有 1–15 秒延迟，仅适合监控，不适合高频交易。

---

## 二、时效性

| 数据类型 | 更新频率 | T+0/T+1 |
|---------|----------|---------|
| **日线收盘价** | 每日收盘后 | T+0（当日收盘后约 1 小时内更新） |
| **实时价** | 秒级/分钟级 | T+0（有 1–15 秒延迟） |
| **基本面** | 季报/年报 | 按披露周期 |

**注意**：盘中扫描时，当日 K 线可能尚未生成，系统使用前一交易日收盘价作为“最新价”。

---

## 三、实时性

| 能力 | 当前实现 | 限制 |
|------|----------|------|
| **实时价获取** | `fetch_realtime()` | 东方财富/新浪接口易限流，失败时回退到日线收盘价 |
| **实时监控** | `python -m src.main watch` | 轮询刷新（默认 10 秒），非 WebSocket |
| **交易所直连** | 未实现 | 需券商 Level-2 或商业 API |

**建议**：盘中监控用 `watch` 命令即可；若需毫秒级延迟，需接入券商或 Wind 等商业数据。

---

## 四、AI 选股

| 功能 | 命令 | 说明 |
|------|------|------|
| **从股池筛选** | `python -m src.main pick` | 从人气榜等股池中由 AI 分析并筛选 |
| **加入自选** | `python -m src.main pick --add` | 将 AI 选中标的加入 watchlist.txt |
| **自选股扫描** | `python -m src.main scan` | 对自选股做 AI 分析 |

**股池来源**：`hot`（人气榜，依赖东方财富）/ `watchlist`（自选股）。东方财富限流时自动回退到自选股。

---

## 五、新增命令速查

```bash
# AI 选股（从人气榜取 20 只分析）
python -m src.main pick

# AI 选股并加入自选
python -m src.main pick --add

# 实时监控自选股（每 10 秒刷新）
python -m src.main watch

# 监控指定股票
python -m src.main watch --symbols 000001,600519 --interval 5

# 自选股管理
python -m src.main watchlist list
python -m src.main watchlist add 000001
python -m src.main watchlist remove 600519
```
